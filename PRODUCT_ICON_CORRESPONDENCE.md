## Usage for UI Integration

### Recommended Implementation Steps:

1. **Extract Icons**: Extract the `All Products Icons.zip` file to a directory accessible by the frontend (e.g., `frontend/public/icons/products/`)

2. **Create Icon Mapping JSON**: Use the `product_icon_correspondence.json` file to create a mapping between product names and icon paths

3. **Update Product Model**: Add an `icon_path` or `icon_url` field to the Product model/database to store the icon file path

4. **Frontend Integration**: 
   - Create a utility function to get icon path by product name
   - Use the icon in product cards, lists, and detail views
   - Fallback to Lucide React icons if icon file not found

### Example Frontend Code:

```typescript
// utils/productIcons.ts
import iconMapping from '../data/product-icon-mapping.json'

export function getProductIconPath(productName: string): string | null {
  const normalizedName = productName.toLowerCase().trim()
  const mapping = iconMapping.find(m => 
    m.icon_product_name.toLowerCase() === normalizedName
  )
  return mapping ? `/icons/products/${mapping.icon_file_name}` : null
}

// Usage in component
const iconPath = getProductIconPath(product.display_name)
{iconPath ? (
  <img src={iconPath} alt={product.display_name} className="w-8 h-8" />
) : (
  <ProductIcon iconName="Package" />
)}
```

## Notes

- Icon file names match product display names from the zip file
- Some products may have multiple icon variants (@2x, @4x) - prefer SVG for scalability
- Icon categories roughly correspond to database categories but may need manual mapping
- Product names in the database may differ slightly from icon file names - use fuzzy matching or manual mapping for best results
