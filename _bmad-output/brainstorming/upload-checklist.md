# Material Upload Checklist & Instructions

## Purpose

This checklist guides you through uploading materials to the central repository. Follow these steps to ensure materials are uploaded correctly and organized properly.

---

## Pre-Upload Checklist

**Before uploading any material, verify:**

- [ ] Material follows naming convention: `[Material Type]_[Product/Universe]_[Version]_[Date]`
- [ ] Material is the current/latest version (check version number or date)
- [ ] Material is complete (not a draft unless marked as such)
- [ ] You know which folder it belongs in (refer to folder structure guide)
- [ ] Material inventory spreadsheet entry is updated with current information
- [ ] File format is correct (PDF, PPTX, DOCX, etc.)

---

## Upload Process: Step-by-Step

### Step 1: Prepare Material

**Actions:**
1. **Verify Material Name:**
   - Follow naming convention: `[Material Type]_[Product/Universe]_[Version]_[Date]`
   - Example: `Product_Brief_Public_Cloud_v2.3_2026-02-01.pdf`
   - Remove spaces or use underscores
   - Keep names descriptive but concise

2. **Check Material Status:**
   - Is this the current version?
   - Is it complete and ready for use?
   - Does it need updates? (If yes, update before uploading)

3. **Verify File Format:**
   - Customer-facing materials: Prefer PDF for professional appearance
   - Internal materials: Can be PPTX, DOCX, or PDF
   - Ensure file opens correctly

4. **Update Inventory Spreadsheet:**
   - Mark material as "Ready to Upload" in Upload Status column
   - Verify all information is correct

---

### Step 3: Upload Material

**Actions:**

1. **Navigate to Target Folder:**
   - Open central repository
   - Navigate to correct folder path
   - Verify you're in the right location

2. **Upload File:**
   - Drag and drop file into folder, OR
   - Click "Upload" button and select file
   - Wait for upload to complete

3. **Verify Upload:**
   - Check file appears in folder
   - Verify file name is correct
   - Ensure file opens correctly

---

### Step 4: Update Inventory Spreadsheet

**Actions:**

1. **Update Upload Status:**
   - Change from "Ready to Upload" to "Uploaded"
   - Add new location path in "New Location" column
   - Example: `01_Internal_Materials/Product_Briefs/By_Product/Public_Cloud/`

2. **Verify Information:**
   - Check all columns are accurate
   - Update any changed information
   - Add notes if needed

---

### Step 5: Quality Check

**Self-Verification:**

- [ ] File uploaded to correct folder
- [ ] File name follows naming convention
- [ ] File opens correctly
- [ ] Inventory spreadsheet updated
- [ ] Material accessible to intended audience (test if needed)

**If Issues Found:**
- Fix immediately (re-upload, rename, move to correct folder)
- Update inventory spreadsheet
- Note issue in spreadsheet Notes column

## Naming Convention Reference

### Format: `[Material Type]_[Product/Universe]_[Version]_[Date]`

**Examples:**

- `Product_Brief_Public_Cloud_v2.3_2026-02-01.pdf`
- `Sales_Deck_Disaster_Recovery_v1.5_2026-01-15.pptx`
- `Datasheet_Object_Storage_v3.1_2026-02-01.pdf`
- `Sales_Enablement_Deck_Private_Cloud_v2.0_2026-01-20.pptx`
- `Product_Catalog_All_Products_v1.0_2026-02-01.pdf`

**Rules:**
- Use underscores, not spaces
- Material Type: Short form (Product_Brief, Sales_Deck, Datasheet)
- Product/Universe: Full name or abbreviation
- Version: v#.# format or date
- Date: YYYY-MM-DD format
- File extension: .pdf, .pptx, .docx

## Troubleshooting Guide

### **Issue: Can't find correct folder**
- **Solution:** Check folder structure guide, create folder if needed
- **Help:** Contact Director or check team channel

### **Issue: Upload fails or is slow**
- **Solution:** Check file size, try smaller file or compress
- **Help:** Contact IT support if persistent

### **Issue: Don't have upload permissions**
- **Solution:** Contact Director to grant permissions
- **Help:** Verify you're logged in with correct account

### **Issue: File name doesn't follow convention**
- **Solution:** Rename file before uploading
- **Help:** Refer to naming convention guide

### **Issue: Not sure which folder to use**
- **Solution:** Check material type and audience (internal vs customer-facing)
- **Help:** Ask Director or team channel for guidance

### **Issue: Material covers multiple products**
- **Solution:** Upload to primary product folder, note in inventory spreadsheet
- **Help:** Consider creating use case folder if appropriate

## Upload Progress Tracking

**Update Inventory Spreadsheet:**

1. **Initial State:** Upload Status = "Not Started"
2. **Prepared:** Upload Status = "Ready to Upload"
3. **Uploaded:** Upload Status = "Uploaded"
4. **Verified:** Upload Status = "Verified" (after quality check)

**Track Your Progress:**
- Update spreadsheet after each upload
- Note any issues in Notes column
- Flag materials needing attention
