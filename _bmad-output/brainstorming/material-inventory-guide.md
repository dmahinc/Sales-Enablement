# Material Inventory Spreadsheet Guide

## Purpose

This spreadsheet helps you inventory all materials you own or have created before uploading them to the central repository. Completing this inventory will help us:
- Identify all materials that need to be uploaded
- Prioritize upload order
- Identify gaps (missing materials)
- Track upload progress
- Ensure nothing is missed

---

### Step 1: Download/Copy the Template

## Column Definitions

### **Material Name**
- **What:** The name/title of the material
- **Format:** Use clear, descriptive names
- **Example:** `Product Brief Public Cloud`, `Sales Deck Disaster Recovery`, `Datasheet Object Storage`

### **Material Type**
- **What:** Category of material (choose from list below)
- **Options:**
  - `Product Brief` (Internal)
  - `Sales Enablement Deck` (Internal)
  - `Product Portfolio` (Internal)
  - `Sales Deck` (Customer-Facing)
  - `Datasheet` (Customer-Facing)
  - `Product Catalog` (Customer-Facing)
  - `Persona` (Shared Asset)
  - `Content Block` (Shared Asset)
  - `Template` (Shared Asset)
  - `Other` (specify in Notes)

### **Product/Universe**
- **What:** Which product or universe this material is for
- **Format:** Product name or Universe name
- **Example:** `Public Cloud`, `Private Cloud`, `Disaster Recovery`, `Business Continuity`
- **Note:** If material covers multiple products, list primary product or "Multiple"

### **Current Location**
- **What:** Where the material currently lives (full path if possible)
- **Format:** File path, SharePoint link, Google Drive link, etc.
- **Example:** 
  - `C:\Documents\Product Briefs\Public Cloud\`
  - `SharePoint\Sales Materials\Public Cloud\`
  - `Google Drive\Materials\Datasheets\`
  - `OneDrive\Product Marketing\`

### **Owner**
- **What:** Your name (the PMM who owns/created this material)
- **Format:** Your full name
- **Example:** `John Doe`, `Jane Smith`

### **Last Updated**
- **What:** Date when material was last modified
- **Format:** YYYY-MM-DD
- **Example:** `2026-01-15`, `2025-12-10`
- **Note:** If unsure, use approximate date

### **Status**
- **What:** Current state of the material
- **Options:**
  - `Current` - Material is up-to-date and accurate
  - `Outdated` - Material needs updates (product changed, info stale)
  - `Missing` - Material should exist but doesn't (gap to fill)
  - `Draft` - Material is work-in-progress
  - `Deprecated` - Material is no longer used (archive)

### **Version**
- **What:** Version number or identifier (if you use versioning)
- **Format:** Version number, date, or identifier
- **Example:** `v2.3`, `2026-01`, `v1.5`
- **Note:** If you don't use versioning, leave blank or use "N/A"

### **File Format**
- **What:** File type/extension
- **Options:**
  - `PDF`
  - `PPTX` (PowerPoint)
  - `DOCX` (Word)
  - `XLSX` (Excel)
  - `Other` (specify in Notes)

### **Language**
- **What:** Language of the material
- **Options:** `English`, `French`, `German`, `Spanish`, `Other` (specify)
- **Note:** If multiple languages exist, create separate rows

### **Audience**
- **What:** Who the material is intended for
- **Options:**
  - `Internal` - For sales teams, pre-sales, customer care (internal use)
  - `Customer-Facing` - For customers/prospects
  - `Shared Asset` - For PMM team (personas, templates, content blocks)

### **Notes**
- **What:** Additional information about the material
- **Format:** Free text
- **Examples:**
  - "Complete with all sections"
  - "Needs update for new features"
  - "Includes new pricing"
  - "Missing use cases section"
  - "Used heavily by sales team"

### **Upload Priority**
- **What:** How urgently this material should be uploaded
- **Options:**
  - `High` - Customer-facing materials, frequently used materials, Product Briefs
  - `Medium` - Internal materials, less frequently used
  - `Low` - Archive materials, drafts, deprecated materials
- **Note:** We'll upload High priority first (Week 2), then Medium/Low (Week 3)

### **Upload Status**
- **What:** Track upload progress (you'll update this during upload)
- **Options:**
  - `Not Started` - Initial state
  - `Ready to Upload` - Material prepared, ready to upload
  - `Uploaded` - Material uploaded to central repository
  - `Verified` - Upload verified, location confirmed
- **Note:** Start with "Not Started", update as you progress

### **New Location**
- **What:** Where material will be located in central repository (fill during upload)
- **Format:** Folder path in central repository
- **Example:** `01_Internal_Materials/Product_Briefs/By_Product/Public_Cloud/`
- **Note:** Leave blank until upload phase

## Inventory Process

### Phase 1: Self-Audit (2-3 hours)
1. **Review Your Files:**
   - Check all locations where you store materials
   - Review your email/SharePoint/Drive folders
   - Check shared folders you contribute to

2. **List All Materials:**
   - Add one row per material
   - Don't worry about perfection - we'll refine together
   - Include materials you've created AND materials you maintain

3. **Identify Gaps:**
   - Are there products missing materials?
   - Are there material types missing for your products?
   - Note these in the spreadsheet (Status = "Missing")

4. **Flag Issues:**
   - Mark outdated materials (Status = "Outdated")
   - Note what needs updating in Notes column

### Phase 2: Review & Refinement (1 hour)
1. **Review Your Inventory:**
   - Check for duplicates
   - Verify all products covered
   - Ensure priority levels assigned

2. **Submit to Director:**
   - Share your completed inventory
   - Director will review and identify any gaps

## Common Questions

**Q: What if I don't know the exact location?**
A: Use approximate location or "Unknown - need to find". We'll locate it during upload.

**Q: What if material covers multiple products?**
A: List the primary product or "Multiple" in Product/Universe column, note details in Notes.

**Q: What if I'm not sure if material is current?**
A: Mark as "Outdated" and note in Notes that it needs review. We'll verify during upload.

**Q: What if material exists but I didn't create it?**
A: Include it if you're responsible for maintaining it. Note original creator in Notes if known.

**Q: How detailed should Notes be?**
A: Enough to help during upload. Note any issues, gaps, or special considerations.
