### **Slide 7: Naming Conventions**

**Title:** File Naming Conventions

**Content:**

**Format:** `[Material Type]_[Product/Universe]_[Version]_[Date]`

**Components:**
- **Material Type:** Product_Brief, Sales_Deck, Datasheet
- **Product/Universe:** Public_Cloud, Disaster_Recovery
- **Version:** v2.3, v1.5, or date format
- **Date:** YYYY-MM-DD format
- **Extension:** .pdf, .pptx, .docx

**Examples:**
- `Product_Brief_Public_Cloud_v2.3_2026-02-01.pdf`
- `Sales_Deck_Disaster_Recovery_v1.5_2026-01-15.pptx`
- `Datasheet_Object_Storage_v3.1_2026-02-01.pdf`

**Rules:**
- âœ… Use underscores, not spaces
- âœ… Keep names descriptive but concise
- âœ… Include version and date
- âœ… Use consistent abbreviations

### **Slide 8: Maintenance Best Practices**

**Title:** Maintaining the Repository

**Content:**

**Regular Maintenance:**
- **When Creating New Materials:**
  - Upload immediately to correct folder
  - Follow naming convention
  - Update inventory spreadsheet

- **When Updating Materials:**
  - Upload new version with updated version/date
  - Archive old version if needed
  - Update inventory spreadsheet

- **When Materials Become Outdated:**
  - Move to Archive folder
  - Update inventory spreadsheet
  - Note reason for archiving

**Monthly:**
- Review folder organization
- Clean up duplicates
- Archive deprecated materials

**Quarterly:**
- Review repository effectiveness
- Gather user feedback
- Adjust structure if needed
